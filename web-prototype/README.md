# 战斗陀螺 Web 验证原型

独立的 HTML / Three.js / Tone.js 版本。所有网页版源码、测试和依赖声明均位于
`web-prototype/`，不依赖 Godot 运行时。

## 已实现

- 五层程序化 3D 陀螺：攻击环、核心锁扣、金属配重盘、驱动中轴、轴尖。
- 15 个正式零件和 `3^5 = 243` 种组合。
- 与 Godot `AssemblyCalculator` 对齐的质量、惯量、摩擦、续航、控制、攻击和耐久计算。
- 标准碗形、金属高速、复合材质三张竞技场。
- 确定性固定步长 1v1 模拟、基础 AI、虚拟摇杆和键盘控制。
- 3D 发射器直接操作：拖动模型调方向与倾角，拖动箭头端点调方向和力度。
- Spin Out、Ring Out、Break 和超时判定。
- Tone.js 发射、旋转、碰撞与结算动态音效。
- 碰撞伤害、转速衰减、控制响应和移动速度实时调参。
- 配置、颜色、赏金、地图和调参结果本地保存。
- 白纸、黑色粗线、电蓝与橙色构成的涂鸦式准备界面。
- 固定 `9:16` 竖屏画布，组装模型作为主视觉，信息沿画面边缘分布。
- 组装页默认只显示陀螺；点选部件后才展开改装，其他部件自动半透明。
- 三个独立出战槽位支持左右横扫切换，并分别保存配置和颜色。
- 地图选择使用横向吸附卡片，一屏完成切换。
- 组装模型支持统一环绕视角、滚轮/双指缩放和视角复位。
- 15 个零件的价格、所有权、金币购买、余额不足反馈和性能差值确认。
- 战斗优先的新手训练：木质陀螺首战、固定金币奖励、首次购买和改装验证战。
- `v2` 本地存档保存零件所有权和教程进度，并兼容迁移 `v1` 配置。
- 战斗主题跟随地图切换，并使用近景镜头和实时操控方向反馈。
- 平移速度、最大移动距离和操控力随转速衰减，避免停转前滑步。
- 碰撞会累积失衡，抬高倾角并在严重时触发擦地掉转与控制衰减。
- 碰撞、Ring Out、转速衰退和倾斜危机通过场内效果和音效反馈，不遮挡战斗。
- 摇杆固定在战斗画面底部；陀螺上方只显示实时控制影响与施力方向。
- 已装备零件支持二次 DIY，可直接拖动模型上的蓝色、橙色、红色手柄调整尺寸、
  高度和轮廓起伏。
- DIY 顶栏提供正视、俯视、侧视、实时质量/惯量和 2 / 3 / 4 / 6 边对称延展；
  切换对称方式会保留已有改造。
- DIY 材料使用金币永久解锁，底栏只保留材料列表、保存和取消；质心继续直接标在
  3D 模型上。
- 组装待机时陀螺自动缓慢旋转，相机有轻微动态；操作后立即停止待机动画。
- 出战槽切换使用陀螺滚动入场，部件聚焦使用相机平滑移动。
- 选择不同零件时执行短暂的旧件拆出、新件装入动画。
- 战斗中的双方转速、耐久和时间集中在顶栏；品牌、金币、声音及风险窗口隐藏。

## 启动

```powershell
cd web-prototype
npm install
npm run dev
```

打开 `http://127.0.0.1:5173/`。

```powershell

```

## 操作

- 组装：轻点模型选择具体部件，未选中的部件会变为半透明；轻点空白退出改装。
- 深度 DIY：再次点击已装备零件进入编辑；保存后写入当前出战槽，取消则丢弃草稿。
- 模型塑形：拖蓝色手柄改大小，拖橙色手柄改高度，拖红色手柄改轮廓起伏。
- 视角：在模型或背景上拖动都会环绕整个场景；滚轮或双指缩放。点击其他部位会
  平滑移动镜头并直接切换编辑目标。
- 出战槽：在空白区域横扫，或点击顶部左右触控区域切换三个陀螺。
- 解锁：灰色零件尚未拥有；金色钱币标记表示余额足够，点击后确认购买并自动装备。
- 地图：左右滑动卡片切换竞技场，底部只有一个确认按钮。
- 发射：拖动发射器/陀螺改变方向和倾角；拖动黄色箭头端点改变方向与力度，
  箭头长度即力度。
- 战斗：摇杆固定在屏幕底部；陀螺附近只显示控制影响，桌面仍支持
  `WASD` 或方向键。
- 新手训练：首次访问从木质陀螺战斗开始，可随时点击“跳过引导”退出训练。

## 技术说明

战斗逻辑使用固定 `1/60s` 步长的确定性二维模拟，Three.js 只负责三维表现。
这种结构便于快速调参、录制输入和复盘，也比浏览器刚体直接参与规则结算更适合作为
后续异步 PVP 的验证基础。Tone.js 在首次用户点击发射或声音按钮后启用音频上下文。

零件经济和教程进度位于 `src/core/progression.js`，三个出战槽位位于
`src/core/loadouts.js`；两者都不参与确定性战斗结算。
每个出战槽的 `customizations` 按基础零件 ID 保存 DIY 参数。改造后的质量、
质心和绕自旋轴惯量由 `AssemblyCalculator` 实时计算，其中惯量包含平行轴项。
材料所有权保存在 `ownedMaterialIds`；迁移旧存档时，已被任一配置使用的合法材料
会自动补为已拥有，避免升级后丢失已有改造。
本地存档键为 `spin-core-web-prototype-v2`；旧版
`spin-core-web-prototype-v1` 会在读取时保留金币、配置和已装备零件，并将原配置
迁移到第一个出战槽。

`BattleSimulation` 构造参数支持：

```js
new BattleSimulation({
  // ...
  diagnostics: true,
  logger: (message, telemetry) => console.info(message, telemetry),
});
```

每次有效碰撞会写入 `collisionLog` 并触发 `collision` 事件。`telemetry` 包含双方
碰撞前后的 `tilt`、`spin`、`imbalance`、`durability` 及对应变化量，方便导出后
分析平衡性。日志最多保留最近 200 次碰撞。

当前失衡与 DIY 规则优先在 Web 原型验证，尚未同步到 Godot
`BattleSimulation`。跨运行时快照对齐前，不应把这版 Web 对局作为异步 PVP
正式结算依据。
