# AI 接手说明：Web 直接操作界面、零件经济与新手训练

更新时间：2026-07-19

## 当前状态

`web-prototype/` 已形成可完整游玩的新手闭环：

1. 首次访问直接进入标准竞技场，使用木质训练陀螺。
2. 玩家自行调节发射参数，并通过摇杆探索战斗。
3. 首战无论胜负固定获得 180 金币。
4. 结算后进入组装界面，购买并自动装备首个零件。
5. 使用新零件完成第二战后结束训练，开放自由组装和地图选择。

训练卡始终提供“跳过引导”，跳过后立即返回组装并解除导航限制。

`progression.js` 负责所有权、购买、奖励和教程状态，不能并入
`battle-simulation.js`，后者必须继续保持确定性。`loadouts.js` 只管理本地出战槽、
颜色和旧存档迁移，也不能把 UI 状态写入模拟。

## 组装交互

- 模型尺寸根据默认相机下的屏幕投影动态拟合。
- 默认不选择部件，只显示当前陀螺和弱化的相邻陀螺。
- 点击模型：射线检测具体部件并展开对应的三个零件选项。
- 选中部件保持实体，其他四个部件降为 20% 透明度。
- 点击空白区域退出改装并恢复全部实体显示。
- 在模型或背景上拖动都环绕相机，不再单独旋转模型。
- 空白区域横扫或顶部左右触控区可切换三个独立出战槽位。
- 滚轮或双指：连续缩放。
- “重置视角”：恢复相机默认构图。

浏览器已验证部件聚焦/退出、统一相机旋转、横扫换陀螺和视角复位。

## 地图、发射与战斗 HUD

- 地图页使用横向 `scroll-snap` 卡片，当前居中卡片与 Three.js 场地同步。
- 地图页底部只有“准备发射”一个主按钮。
- 战斗准备阶段显示程序化 3D 发射器。
- 发射高度、方向和倾角会实时改变发射器及陀螺姿态。
- 发射高度已经进入确定性模拟：略微影响初始平移速度、自旋和倾斜。
- 发射后摇杆根据玩家陀螺的屏幕投影位置跟随移动，不再使用底部控制窗口。

## 存档与迁移

当前存档键：

```text
spin-core-web-prototype-v2
```

新增字段：

```text
version
ownedPartIds
loadouts
activeLoadoutIndex
tutorial.stage
tutorial.completed
tutorial.firstRewardClaimed
```

读取旧 `v1` 存档时会保留金币、配置、颜色、地图和调参结果。旧单配置会进入
第一个出战槽，另外两个槽位使用基础配置。所有槽位中已装备的有效零件都会自动加入
所有权，避免迁移后丢失装备。

当前存档版本仍为 `2`。新增槽位字段时不要只依赖 `version` 判断：`migrateLoadouts`
必须继续接收缺少 `loadouts` 的已有 v2 存档。

教程阶段：

```text
first_battle
buy_first_part
second_battle
complete
```
